## 커피 전문점 도메인
### 커피 주문

1. 메뉴판 : 
 - 아메리카노 1500원
 - 카푸치노 2000원
 - 카라멜 마끼야토 2500원
 - 에스프레소 2500원
2. 등장인물 : 
 - 손님
 - 바리스타
 
### 커피 전문점이라는 세상
1. 객체
- 메뉴판
  - 각각의 메뉴
 - 손님
 - 바리스타
 - 커피
2. 관계
- 손님은 메뉴판에서 커피를 선택한다
 - 손님 -> 메뉴판 
- 손님은 바리스타에게 커피를 주문한다
 - 손님 -> 바리스타
- 바리스타는 커피를 만든다.
 - 바리스타 -> 커피
- 메뉴판에는 메뉴항목이 있다.
 - 메뉴판 <- 메뉴항목 (합성관계)

 ## 설계하고 구현하기
 ### 커피를 주문하기 위한 협력 찾기
 - 훌륭한 *협력을 설계*하는 것이 목표
 - 메세지 선택 후 객체에게 전달한다
 
 1. 첫번째 메세지 :  커피를 주문하라
  - + 메뉴 이름
 2. 적절현 객체를 찾기
  - 준비해둔 도메인 모델에서 찾기
  - 손님 객체 
 3. 손님이 할 수 없는 일을 누구에게 도움 받을 것인가?
  - 메세지 : 메뉴 항목을 찾아라
  - 메뉴판 객체 
   - 현실을 은유함.
   - 현실보다 더 많은 일을 할 수 있는 존재
 4. 커피를 제조하라
  - 바리스타 객체
 
 ### 인터페이스 정리하기
 - 인터페이스 : 어떤 객체가 어떤 메세지를 수신하는 지를 정리한 것.
 
 ### 구현하기
- 손님 객체에서 커피를 주문하기 위해서 접근해야하는 객체는?
 - 메뉴판, 바리스타
- 어떻게 전달할 것인가?
 - 손님의 인터페이스를 변경하여 메뉴와 바리스타 객체를 전달받는다
- 메뉴를 선택하기 위해 메뉴판은 무엇이 필요한가?
 - MenuItem을 리스트로 갖는다
 - MenuItem은 어떤 정보를 갖는가? 가격, 메뉴명
 - MenuItem을 리스트 속성으로 추가한다

*이러한 방식으로 구현하는 것* 
- 도메인 구성 -> 인터페이스 작성 -> 코드로 빠르게 구현 후 설계가 잘되었는지 평가, 확인
- TDD는 테스트 방식으로 테스트 코드로서 이 방법을 실행하는 것
